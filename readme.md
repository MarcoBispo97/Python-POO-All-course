## Overview

This repository is a collection of notes, code examples, and projects that demonstrate the core principles of Object-Oriented Programming in Python. It is structured to help learners and developers reinforce their knowledge of OOP while exploring practical implementations.

## Key Concepts Covered

- **Classes and Objects**: Understanding the blueprint of objects.
- **Attributes and Methods**: Differentiating between instance and class attributes.
- **Encapsulation**: Hiding the internal state and requiring all interactions to be performed through methods.
- **Abstraction**: Simplifying complex systems by modeling classes appropriate to the problem.
- **Inheritance**: Reusing code and creating hierarchical relationships.
- **Polymorphism**: Designing objects to share behaviors, allowing for flexible and reusable code.
- **Docstrings**: Writing clear and concise documentation for classes and methods.

## Folder Structure

The repository is organized into the following sections:

- **[S1_Welcome/](S1_Welcome/book1.ipynb)**: Introduction to the course.
- **[S2_Key_Components/](S2_Key_Components/book2.ipynb)**: Overview of OOP, its advantages, and important terminology.
- **[S3_Classes_the_Blueprints_of_OOP/](S3_Classes_the_Blueprints_of_OOP/S3_note.ipynb)**: Basics of classes and objects, including naming conventions and class headers.
- **[S4_Instances_and_Instances_attributes/](S4_Instances_and_Instances_attributes/S4_note.ipynb)**: Understanding instances, instance attributes, and the `__init__` method.
- **[S5_Class_attributes_define_shared/](S5_Class_attributes_define_shared/S5_note.ipynb)**: Exploring class attributes and their shared nature across instances.
- **[S6_Encapsulation_and_Abstraction/](S6_Encapsulation_and_Abstraction/S6_note.ipynb)**: Principles of encapsulation and abstraction, including public and non-public attributes.
- **[S7_Properties_getters_setters/](S7_Properties_getters_setters/S7_note.ipynb)**: Using properties, getters, and setters to manage and protect instance attributes.
- **[S8_Methods_add_functionality/](S8_Methods_add_functionality/S8_note.ipynb)**: Adding functionality to classes with instance, class, and static methods.
- **[S9_project_build_a_dice_game/](S9_project_build_a_dice_game/S9_note.ipynb)**: A mini-project to build a dice game, demonstrating aggregation and relationships between classes.
- **[S10_Objects_in_memory/](S10_Objects_in_memory/S10_note.ipynb)**: Understanding how objects are stored in memory, references, and the `is` operator.
- **[S11_Aliasing_Mutation_Cloning/](S11_Aliasing_Mutation_Cloning/S11_note.ipynb)**: Exploring aliasing, mutation, and cloning of objects.
- **[S12_Build_a_Tic_tac_toe_game_with_oop/](S12_Build_a_Tic_tac_toe_game_with_oop/)**: A complete Tic-Tac-Toe game built with OOP principles.
- **[S13_Inheritance_in_Python_Attributes/](S13_Inheritance_in_Python_Attributes/S13_note.ipynb)**: Understanding inheritance of attributes and its advantages.
- **[S14_Inheritance_in_python_Methods/](S14_Inheritance_in_python_Methods/S14_note.ipynb)**: Method inheritance, overriding, and polymorphism.
- **[S15_Working_with_multiple_files/](S15_Working_with_multiple_files/S15_note.ipynb)**: Organizing projects with modules, files, and import statements.
- **[S16_Docstrings/](S16_Docstrings/S16_note.ipynb)**: Best practices for writing docstrings, including PEP 257 conventions and their use in generating documentation.

Each folder contains notes, code examples, and exercises to reinforce the concepts.

## How to Use

1. Clone the repository:
   ```bash
   git clone https://github.com/MarcoBispo97/Python-POO-All-course.git
   ```
2. Navigate to the project directory:
   ```bash
   cd Python-POO-All-course
   ```
3. Set up a virtual environment:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```
4. Explore the folders and run the Python scripts to practice the concepts.
